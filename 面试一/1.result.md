1.  var 会提升，let 和 const 不会提升
    比如

```javascript
console.log(a); // undefined
var a = 10;
```

var 没有块级作用域，let 和 const 有块级作用域
比如

```javascript
{
  let a = 10;
  const b = 20;
  console.log(a); // 10
  console.log(b); // 20
}
console.log(a); // ReferenceError: a is not defined
console.log(b); // ReferenceError: b is not defined
```

var 重复声明会覆盖之前的声明，编译不会报错
let、const 重复声明会报错
比如

```javascript
var a = 10;
var a = 20;
console.log(a); // 20
```

const 不支持修改，let、var 支持修改

2. 什么是原型链？
   每个对象都有一个原型对象，原型对象也有自己的原型对象，以此类推，形成一个原型链。
   比如

```javascript
function Person() {
  this.name = "张三";
}
const p = new Person();
console.log(p.name); // 张三
console.log(p.__proto__); // Person {}
console.log(p.__proto__.__proto__); // Object {}
```

原型链的作用就是实现继承，一个对象直接可以使用它原型链上有的属性和方法（先从自身找，找不到再去原型链上一级一级往上找）
proto 是每个对象都有的属性，它指向它的原型对象
prototype 是每个函数都有的属性，它指向这个函数作为构造函数时 new 出来的新对象的原型对象
顺带一提：js 中万物皆对象，也就是说函数也是对象，所以函数也有原型对象

```javascript
// 原型链实现继承
function SuperType() {
  this.name = "张三";
}
function SubType() {
  this.age = 18;
}
SubType.prototype = new SuperType();
const p = new SubType();
console.log(p.name); // 张三
console.log(p.__proto__); // Person {}
console.log(p.__proto__.__proto__); // Object {}

// ES6继承
class SuperType {
  constructor(name) {
    this.name = name;
  }
}
class SubType extends SuperType {
  constructor(name, age) {
    super(name);
    this.age = age;
  }
}
const p = new SubType("张三", 18);
console.log(p.name); // 张三
console.log(p.__proto__); // SubType {}
console.log(p.__proto__.__proto__); // SuperType {}
console.log(p.__proto__.__proto__.__proto__); // Object {}
```

4.闭包
闭包是函数可以访问其外部函数作用域的变量，即使外部函数已经执行完毕。
闭包应用场景
柯理化函数、闭包可以实现私有变量和方法（比如防抖和节流）
闭包可能会导致内存泄漏，因为闭包会引用外部函数的变量，所以如果闭包没有被释放，外部函数的变量就不会被释放。

5.防抖和节流

```javascript
// 防抖
function debounce(fn, delay) {
  let timer = null;
  return function (...args) {
    if (timer) {
      clearTimeout(timer);
    }
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 节流
function throttle(fn, delay) {
  let timer = null;
  return function (...args) {
    if (timer) {
      return;
    }
    timer = setTimeout(() => {
      fn.apply(this, args);
      timer = null;
    }, delay);
  };
}
```

防抖是触发一次后，在指定时间内如果再次触发，会重新计算时间。
节流是触发一次后，在指定时间内无论触发多少次，都只执行第一次。

二、React
1.React 是通过修改状态->重新渲染组件->更新 DOM 来实现页面的更新的。
直接修改状态不会触发 Fiber 的更新，需要通过 setState 来修改状态，setState 会触发 Fiber 的更新。
setState(prev => prev + 1)解决了闭包状态的问题，因为它是异步的，所以可以确保每次修改状态都是基于最新的状态。
在闭包下 setState(prev + 1) 是无效的，因为 setState 是异步的，所以在闭包中 setState(prev + 1) 不会基于最新的状态，而是基于初始状态。
setState(prev => prev + 1)适用闭包、异步调用、批量更新状态。

2.useEffect 依赖数组是作为响应式检查，只有当依赖数组中的变量发生变化时，才会再次执行 useEffect 中的函数。
如果依赖数组为空，那么 useEffect 中的函数只会在组件挂载时执行一次。
如果依赖数组省略，那么 useEffect 中的函数会在每次组件渲染时执行。
如果依赖数组中存在变量，那么 useEffect 中的函数会在变量发生变化后且组件挂载后执行。

3.虚拟 DOM 是 React 中一种优化性能的机制。
它是一个轻量级的 JavaScript 对象，用来描述真实 DOM 的结构。
当组件的状态或属性发生变化时，React 会根据新的状态或属性创建一个新的虚拟 DOM。
然后 React 会对比新旧虚拟 DOM 的差异，只更新发生变化的部分，而不是全部重新渲染。
Diff 算法是 React 中用来对比新旧虚拟 DOM 的差异的算法。
它会遍历新旧虚拟 DOM 树，对比每个节点的差异，最后生成一个补丁对象，用来描述需要更新的部分。
key 是 React 中用来标识节点的唯一标识符。
当节点发生变化时，React 会根据 key 来判断是否需要更新节点。
如果 key 相同，那么 React 会认为节点没有发生变化，直接复用节点。
如果 key 不同，那么 React 会认为节点发生了变化，需要重新渲染节点。
所以 key 是 React 中用来优化性能的重要机制。

React 组件通信有以下几种方式：

1. 父组件向子组件通信：通过 props 传递数据。
2. 子组件向父组件通信：通过回调函数传递数据。
3. 跨级组件通信：通过 context 传递数据。
4. 兄弟组件通信：通过共同的父组件传递数据。

4.自定义 Hook useLocalStorage

```javascript
import { useState } from "react";

function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.log(error);
      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      setStoredValue(value);
      window.localStorage.setItem(key, JSON.stringify(value));
    } catch (error) {
      console.log(error);
    }
  };

  return [storedValue, setValue];
}

const [storedValue, setStoredValue] = useLocalStorage("storedValue", []);
```

1.Webpack 的核心概念有：
入口（entry）：Webpack 开始打包的文件，默认是 ./src/index.js。
出口（output）：Webpack 打包输出的文件，默认是 ./dist/main.js。
加载器（loader）：Webpack 用来处理非 JavaScript 文件的模块，比如 CSS、图片等。
插件（plugin）：Webpack 用来扩展 Webpack 功能的模块，比如压缩代码、优化性能等。
模式（mode）：Webpack 打包的模式，有 development 和 production 两种模式。
development 模式：打包速度快，代码没有压缩，适合开发环境。
production 模式：打包速度慢，代码压缩后，适合生产环境。

loader 和 plugin 的区别：
loader：用来处理非 JavaScript 文件的模块，比如 CSS、图片等。loader 会在模块解析阶段处理文件，将其转换为 Webpack 可以识别的模块。
plugin：用来扩展 Webpack 功能的模块，比如压缩代码、优化性能等。plugin 可以参与 webpack 的整个打包过程，比如压缩代码、优化性能等。

1.浏览器的缓存机制分为哪几种？强缓存和协商缓存的区别是什么？304 状态码的触发条件是什么？
浏览器缓存机制分为以下几种： 1.强缓存 2.协商缓存 3.无缓存
强缓存和协商缓存的区别：
强缓存：浏览器会根据响应头中的 Cache-Control 或 Expires 字段来判断是否缓存资源。如果缓存有效，浏览器会直接从缓存中读取资源，不会向服务器发送请求。
协商缓存：浏览器会根据响应头中的 ETag 或 Last-Modified 字段来判断是否缓存资源。如果缓存有效，浏览器会向服务器发送请求，服务器会根据请求头中的 If-None-Match 或 If-Modified-Since 字段来判断是否返回 304 状态码。
304 状态码的触发条件：
浏览器会根据协商缓存的规则来判断是否返回 304 状态码。如果缓存有效，服务器会返回 304 状态码，浏览器会从缓存中读取资源。

2.从输入 URL 到页面渲染的完整流程？

1. 浏览器会根据 URL 来解析域名，获取 IP 地址。
2. 浏览器会根据 IP 地址来建立 TCP 连接。
3. 浏览器会根据 TCP 连接来发送 HTTP 请求。
4. 服务器会根据 HTTP 请求来处理请求，返回 HTTP 响应。
5. 浏览器会根据 HTTP 响应来解析 HTML 代码。
6. 浏览器会根据 HTML 代码来构建 DOM 树。
7. 浏览器会根据 DOM 树来解析 CSS 代码。
8. 浏览器会根据 CSS 代码来构建 CSSOM 树。
9. 浏览器会根据 DOM 树和 CSSOM 树来构建渲染树。
10. 浏览器会根据渲染树来布局页面。
11. 浏览器会根据布局信息来绘制页面。

3.什么是跨域？跨域的解决方法有哪些？
跨域：是一种安全策略，用于限制一个域的页面访问另一个域的资源。
解决方法：

1. JSONP：利用 script 标签的 src 属性来跨域请求数据。
2. CORS：服务器端设置 Access-Control-Allow-Origin 头来允许跨域请求。
3. 代理服务器：将跨域请求发送到同源的代理服务器，由代理服务器来请求目标服务器。

4.如何优化前端页面的首屏加载速度？列举至少 5 种方案。

7. 减少 DOM 操作次数：避免频繁操作 DOM，减少 DOM 操作次数。
