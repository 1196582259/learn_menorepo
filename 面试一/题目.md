字节跳动前端工程师面试题（含答案解析）
一、基础概念题（考察前端核心理论掌握程度）
请详细解释 HTTP 1.0 1.1、HTTP 2、HTTP 3 的主要区别，以及它们在实际项目中对性能的影响。
什么是闭包？闭包的形成条件是什么？请举例说明闭包在实际开发中的应用场景，以及使用闭包可能带来的问题。
阐述 JavaScript 中的原型链机制，包括 Object.prototype 和 Function.prototype 之间的关系，并用代码示例说明原型继承的实现方式。
请对比 var、let、const 的区别，包括变量提升、作用域、重复声明、修改值等方面，并用代码示例验证。
什么是浏览器的重排（Reflow）和重绘（Repaint）？列举至少 5 种会触发重排的操作，以及在项目中如何减少重排和重绘以优化性能。
二、DOM 与 BOM 操作题（考察浏览器交互能力）
请用原生 JavaScript 实现一个函数，批量给页面中所有 class 为 "item" 的元素添加点击事件，点击后输出该元素的文本内容，要求避免事件委托的冒泡问题。
阐述浏览器的事件流机制（捕获阶段、目标阶段、冒泡阶段），并实现一个案例：给父元素和子元素都绑定点击事件，通过代码控制只触发子元素的事件，不触发父元素的事件。
如何获取 DOM 元素的宽高（包括内容宽高、padding 宽高、border 宽高、margin 宽高）？请分别写出对应的 API 及获取方式，并说明不同 API 的兼容性。
请解释 BOM 中的 location 对象和 history 对象的常用属性及方法，并用代码示例实现页面的前进、后退、刷新，以及跳转到指定 URL 并记录历史记录的功能。
三、框架与工程化题（考察主流技术栈应用能力，以 React 为例）
详细阐述 React 的虚拟 DOM 原理，包括虚拟 DOM 的创建、Diff 算法的核心逻辑（如 Key 的作用、同层比较策略），以及虚拟 DOM 如何提升页面渲染性能。
对比 React 中的类组件和函数组件，包括状态管理、生命周期、代码复用方式（如 HOC、Render Props、Hook）等方面，说明为什么现在更推荐使用函数组件。
解释 React Hook 的设计初衷，详细说明 useState、useEffect、useMemo、useCallback 的用法及注意事项，并用 Hook 实现一个具有防抖功能的输入框组件。
什么是前端工程化？前端工程化包含哪些方面（如构建工具、代码规范、模块化、自动化测试等）？以 Webpack 为例，说明如何配置一个基础的 React 项目构建环境（至少包含入口、出口、Loader、Plugin 配置）。
阐述 React 状态管理方案（如 Context + useReducer、Redux、MobX、Zustand 等），对比 Redux 和 Zustand 的优缺点，并用 Zustand 实现一个简单的计数器状态管理案例。
四、性能优化题（考察项目性能把控能力）
从网络层面、渲染层面、代码层面分别阐述前端性能优化的方案，每个层面至少列举 4 种具体措施，并说明其优化原理。
什么是懒加载和预加载？它们的应用场景有哪些？请用原生 JavaScript 实现图片的懒加载功能，要求兼容移动端。
阐述浏览器的缓存机制（包括强缓存、协商缓存），说明它们的触发条件、缓存位置（如 Memory Cache、Disk Cache），以及在项目中如何合理设置缓存策略以减少重复请求。
什么是首屏加载时间、白屏时间、首次内容绘制（FCP）、最大内容绘制（LCP）？如何通过 Chrome 开发者工具查看这些指标？并给出至少 3 种优化首屏加载时间的具体方案。
五、算法与编程题（考察逻辑思维与代码实现能力）
实现一个深拷贝函数，要求能正确拷贝基本数据类型、引用数据类型（如对象、数组、函数、正则、日期等），并处理循环引用问题。
给定一个无序数组，实现一个函数找出数组中第 k 大的元素，要求时间复杂度尽可能低（至少给出两种解法，如排序法、堆排序法）。
实现一个防抖函数（debounce）和节流函数（throttle），并说明它们的区别及应用场景（如搜索输入防抖、滚动事件节流），最后用代码验证其效果。
给定一个字符串，判断该字符串是否是回文串（只考虑字母和数字字符，忽略字母的大小写），例如：输入 "A man, a plan, a canal: Panama"，输出 true；输入 "race a car"，输出 false。
六、场景题（考察实际问题解决能力）
在 React 项目中，当父组件频繁渲染时，如何避免子组件不必要的重渲染？请列举至少 3 种解决方案，并说明每种方案的适用场景。
项目中遇到跨域问题，你知道哪些跨域解决方案？请详细说明每种方案的原理、配置方式及适用场景（如 CORS、JSONP、代理服务器、postMessage 等）。
在移动端前端开发中，如何解决常见的适配问题（如不同屏幕尺寸、Retina 屏、刘海屏）？请说明具体的适配方案（如 rem、vw/vh、Flexbox、Grid、媒体查询等）及实践经验。
假设你负责的前端项目上线后，用户反馈部分页面在 Safari 浏览器中出现样式错乱和功能失效的问题，你会如何排查和解决这个问题？请描述具体的排查步骤和可能的解决方案。
