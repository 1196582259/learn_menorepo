## 一、JavaScript 基础（30 分）

### 1. `var`、`let`、`const` 的区别

| 特性     | var                     | let                        | const                           |
| -------- | ----------------------- | -------------------------- | ------------------------------- |
| 作用域   | 函数作用域 / 全局作用域 | 块级作用域                 | 块级作用域                      |
| 变量提升 | 存在（值为`undefined`） | 暂时性死区（不可提前访问） | 暂时性死区                      |
| 重复声明 | 允许                    | 不允许                     | 不允许                          |
| 值修改   | 允许                    | 允许                       | 引用不可改，对象 / 数组内部可改 |
| 全局挂载 | 挂载到`window`          | 不挂载                     | 不挂载                          |

**关键示例**：

```
const obj = { a: 1 };

obj.a = 2; // 可行

// obj = { b: 2 }; // 报错（修改引用）
```

### 2. 原型链 & 继承

#### 核心概念

- 原型链：对象通过 `__proto__` 层层指向原型，直到 `Object.prototype.__proto__ = null`，实现属性继承

- 关系：`实例.__proto__ === 构造函数.prototype`

- `__proto__`：所有对象都有，指向原型；`prototype`：所有函数都有，指向实例原型

#### 两种继承实现

```javascript
// 1. 原型链继承（修复引用共享问题）

function SuperType() {
  this.name = "张三";
}

SuperType.prototype.sayName = () => console.log(this.name);

function SubType() {
  this.age = 18;
}

SubType.prototype = new SuperType();

SubType.prototype.constructor = SubType;

// 2. ES6 Class 继承（推荐）

class SuperType {
  constructor(name) {
    this.name = name;
  }

  sayName() {
    console.log(this.name);
  }
}

class SubType extends SuperType {
  constructor(name, age) {
    super(name); // 必须调用

    this.age = age;
  }
}
```

### 3. 事件循环（宏任务 / 微任务）

#### 执行顺序

1. 执行同步代码 → 2. 清空所有微任务 → 3. 浏览器渲染 → 4. 执行 1 个宏任务 → 重复

#### 任务类型

- 宏任务：`setTimeout`/`setInterval`、UI 渲染、I/O、`script` 整体

- 微任务：`Promise.then`、`async/await`、`queueMicrotask`

#### 代码输出

```javascript
// 输入代码

console.log("A");

setTimeout(() => console.log("B"), 0);

Promise.resolve().then(() => console.log("C"));

new Promise((resolve) => {
  console.log("D");
  resolve();
}).then(() => console.log("E"));

console.log("F");

// 输出：A → D → F → C → E → B
```

### 4. 闭包

- 定义：函数访问词法作用域变量，即使在作用域外执行

- 应用：私有变量、防抖节流、柯里化

- 内存泄漏：未清理闭包引用（如未销毁的定时器）→ 解决方案：手动清理

### 5. 防抖 & 节流

#### 区别与实现

```javascript
// 防抖（重复触发重置延迟）

function debounce(fn, delay, immediate = false) {
  let timer = null;

  return function (...args) {
    if (timer) clearTimeout(timer);

    if (immediate && !timer) fn.apply(this, args);

    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流（指定时间内最多执行一次）

function throttle(fn, delay) {
  let lastTime = 0;

  return function (...args) {
    const now = Date.now();

    if (now - lastTime >= delay) {
      fn.apply(this, args);

      lastTime = now;
    }
  };
}
```

- 防抖场景：搜索输入、按钮防重点击

- 节流场景：滚动加载、窗口 resize

## 二、React 核心（30 分）

### 1. `useState` 更新机制

- 直接修改状态不触发重渲染，需通过更新函数

- 两种更新方式：

  - 值更新：`setCount(count + 1)`（不依赖前状态）

  - 函数式更新：`setCount(prev => prev + 1)`（闭包 / 异步 / 批量更新场景）

### 2. `useEffect` 依赖数组

| 依赖数组 | 执行时机                       |
| -------- | ------------------------------ |
| `[]`     | 组件挂载时执行一次，卸载时清理 |
| 省略     | 每次渲染后都执行               |
| `[a, b]` | a/b 变化或组件挂载后执行       |

- 避免无限循环：依赖数组包含所有用到的变量，函数用 `useCallback` 缓存

### 3. 虚拟 DOM & Diff 算法

- 虚拟 DOM：轻量 JS 对象描述真实 DOM，减少 DOM 操作、支持跨平台

- Diff 算法核心：

1. 同层比较（不跨层级）

2. key 是节点唯一标识（避免用索引）

3. 节点类型不同则销毁重建

### 4. 组件通信方式

| 场景   | 实现方式                           |
| ------ | ---------------------------------- |
| 父子   | Props + 回调函数                   |
| 跨层级 | Context API + useContext / Zustand |
| 兄弟   | 父组件中转 / 事件总线              |
| 页面   | 路由参数 /localStorage             |

### 5. 自定义 Hook `useLocalStorage`

```javascript
import { useState } from "react";

function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = localStorage.getItem(key);

      return item
        ? JSON.parse(item)
        : typeof initialValue === "function"
        ? initialValue()
        : initialValue;
    } catch (e) {
      console.error(e);

      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      const val = value instanceof Function ? value(storedValue) : value;

      setStoredValue(val);

      localStorage.setItem(key, JSON.stringify(val));
    } catch (e) {
      console.error(e);
    }
  };

  return [storedValue, setValue];
}
```

### 4. 手写简易 Promise

```javascript
class MyPromise {
  constructor(executor) {
    this.status = "pending";

    this.value = undefined;

    this.reason = undefined;

    this.onFulfilledCallbacks = [];

    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.status === "pending") {
        this.status = "fulfilled";

        this.value = value;

        this.onFulfilledCallbacks.forEach((cb) => cb());
      }
    };

    const reject = (reason) => {
      if (this.status === "pending") {
        this.status = "rejected";

        this.reason = reason;

        this.onRejectedCallbacks.forEach((cb) => cb());
      }
    };

    try {
      executor(resolve, reject);
    } catch (err) {
      reject(err);
    }
  }

  then(onFulfilled, onRejected) {
    onFulfilled =
      typeof onFulfilled === "function" ? onFulfilled : (val) => val;

    onRejected =
      typeof onRejected === "function"
        ? onRejected
        : (err) => {
            throw err;
          };

    const promise2 = new MyPromise((resolve, reject) => {
      if (this.status === "fulfilled") {
        setTimeout(() => {
          try {
            const x = onFulfilled(this.value);

            resolve(x);
          } catch (err) {
            reject(err);
          }
        }, 0);
      }

      if (this.status === "rejected") {
        setTimeout(() => {
          try {
            const x = onRejected(this.reason);

            resolve(x);
          } catch (err) {
            reject(err);
          }
        }, 0);
      }

      if (this.status === "pending") {
        this.onFulfilledCallbacks.push(() => {
          setTimeout(() => {
            try {
              const x = onFulfilled(this.value);

              resolve(x);
            } catch (err) {
              reject(err);
            }
          }, 0);
        });

        this.onRejectedCallbacks.push(() => {
          setTimeout(() => {
            try {
              const x = onRejected(this.reason);

              resolve(x);
            } catch (err) {
              reject(err);
            }
          }, 0);
        });
      }
    });

    return promise2;
  }
}
```
