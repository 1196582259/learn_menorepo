执行上下文是指代码运行的环境，其中包含了变量对象、作用域链、this 等信息。
执行栈：是一种先进后出的数据结构，用于存储和管理执行上下文。
当代码开始执行时，会创建一个全局执行上下文并压入执行栈中。
每当遇到一个函数调用时，就会创建一个新的函数执行上下文并压入执行栈中。
当函数执行完成后，对应的执行上下文会从执行栈中弹出。

```javascript
function a() {
  function b() {
    console.log("b");
  }
  b();
}
a();
```

以上这段代码的执行过程如下：
全局执行上下文被创建并压入执行栈中。
调用函数 a，创建函数 a 的执行上下文并压入执行栈中。
在函数 a 中调用函数 b，创建函数 b 的执行上下文并压入执行栈中。
函数 b 执行完成，对应的执行上下文从执行栈中弹出。
函数 a 执行完成，对应的执行上下文从执行栈中弹出。

深拷贝就是创建一个新的对象，将原对象的属性值复制到新对象中，如果原对象的属性值是引用类型，那么就递归复制引用类型的属性值。  
 浅拷贝就是创建一个新的对象，如果原类型为基本类型，那么就直接复制值；如果原类型为引用类型，那么就复制引用地址。

深拷贝的实现方法有：

1.  递归拷贝 优点：可以深拷贝任意层级的对象 缺点：如果对象中存在循环引用，会导致栈溢出
2.  JSON.stringify() 和 JSON.parse() 优点：简单易用 缺点：不能深拷贝函数、正则表达式、日期对象等，循环引用会报错
3.  第三方库 lodash.cloneDeep() 优点：功能完善 缺点：依赖第三方库

async/await 是 ES2017 引入的异步编程语法，它基于 Promise 实现。
它的工作原理是等待 Promise 执行完成后，将 Promise 的结果赋值给变量，然后继续执行后面的代码，await 后面的代码会被放到微任务队列中。

```javascript
async function async1() {
  console.log("async1 start");
  await async2();
  console.log("async1 end");
}
async function async2() {
  console.log("async2");
}
console.log("script start");
setTimeout(() => console.log("setTimeout"), 0);
async1();
new Promise((resolve) => {
  console.log("promise1");
  resolve();
}).then(() => console.log("promise2"));
console.log("script end");
/**
 * 这串代码输出：
 * script start
 * async1 start
 * async2
 * promise1
 * script end
 * async1 end
 * promise2
 * setTimeout
 */
```

请实现一个带并发限制的异步任务调度器，要求：① 最多同时执行 N 个任务；② 任务完成后自动执行队列中的下一个任务；③ 支持添加任务和获取所有任务完成后的结果。

```typescript
type Task = () => void & unknown;

class TaskScheduler {
  #limit;
  #queue: Task[];
  #running;
  #results: unknown[];
  constructor(limit: number) {
    this.#limit = limit;
    this.#queue = [];
    this.#running = 0;
    this.#results = [];
  }
  add(task: Task) {
    this.#queue.push(task);
    // 加入一个任务时，要判断是否有空余位置，有则立即执行
    this.run();
  }
  run() {
    // 当运行中的任务数小于限制数时，且队列中还有任务时，执行任务
    while (this.#running < this.#limit && this.#queue.length > 0) {
      const task = this.#queue.shift()!;
      this.#running++;
      let taskResult: Promise<unknown>;
      if (typeof task === "function") {
        try {
          const result = task();
          taskResult = Promise.resolve(result);
        } catch (error) {
          taskResult = Promise.reject(error);
        }
      } else {
        taskResult = Promise.resolve(task);
      }
      taskResult.then((value) => {
        this.#running--;
        // 任务执行完成后，继续执行队列中的任务
        this.run();
      });
      this.#results.push(taskResult);
    }
  }

  getResults() {
    return this.#results;
  }
}
```

this 指向只有 3 种情况：

1. 全局作用域下的 this 指向 window，
2. 调用时确认 this 指向，不能在定义时确认，比如构造函数，虽然构造函数使用 new 关键字新出来了一个对象，但它的 constructor 由新出来的对象调用，所以在 constructor 中的 this 指向新出来的对象
3. 编译时确认 this 指向，不能在运行时确认，箭头函数中的 this 指向定义时的上下文，而不是调用时的上下文

call 方法可以改变函数的 this 指向，它不会生成新的函数，而是直接调用函数。应用场景：调用对象上的方法
apply 方法和 call 方法类似，不同的是 apply 方法接受一个数组作为参数，而 call 方法接受多个参数。
bind 方法可以改变函数的 this 指向，它会生成一个新的函数，而不是直接调用函数。应用场景：调用对象上的方法
